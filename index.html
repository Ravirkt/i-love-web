2 September 2024

Wat heb ik vandaag gedaan?

Vandaag heb ik veel geleerd over HTML. Ik heb verschillende dingen geprobeerd, zoals het toevoegen van lijsten en afbeeldingen op een webpagina. Het was interessant om te ontdekken hoe dat werkt. Ook heb ik met Git gewerkt. Ik heb geleerd hoe ik een repository kan forken en mijn eigen aanpassingen kan doorvoeren. Daarna ben ik verder gegaan met CSS, waarmee ik de opmaak van de pagina kon veranderen. Ik heb verschillende stijlen uitgeprobeerd en gekeken hoe dat de pagina beïnvloedt.

Daily checkout

3 dingen die ik geleerd heb:

- html semantics
- Belang maken van schetsen
- verschillende html-tags

Vandaag heb ik veel geleerd over HTML-semantiek. Ik begrijp nu beter hoe verschillende HTML-tags werken en wat hun rol is in het structureren van een webpagina. Ik heb geleerd hoe je de juiste tags kiest om je content duidelijk en logisch op te bouwen. Daarnaast heb ik ook gewerkt aan het maken van breakdown-schetsen, waarbij ik de indeling van een website op papier zet voordat ik begin met coderen. Dit helpt me om een duidelijk overzicht te krijgen van de opbouw van een pagina en maakt het makkelijker om alles in HTML te vertalen.

2 dingen die ik interesant vind:

- css boxmodel
- Belang maken van schetsen

Twee dingen die ik erg interessant vind, zijn het CSS box-model en breakdown-schetsen. Het box-model helpt me beter te begrijpen hoe elementen op een webpagina worden ingedeeld en hoeveel ruimte ze innemen. Ik vind het leuk om te zien hoe marges, padding, en borders samenwerken om de lay-out vorm te geven. Daarnaast vind ik breakdown-schetsen handig, omdat ik daarmee eerst een duidelijk overzicht van de structuur van een website kan maken voordat ik begin met coderen. Het helpt me om mijn ideeën visueel te ordenen en beter voorbereid te zijn.

4 September 2024

Wat heb ik vandaag gedaan?

Vandaag heb ik geleerd over het maken van een Breakdown schets. Dit heb ik toegepast aan mijn schets. verder heb ik verder gewerkt aan mijn visitekaartje. De styling heb ik aangepast en ik heb basic animaties toegevoegd.

1.Wat is het nut van een breakdown schets Dit maakt duidelijk welke elementen en properties van css, html en javascript je kunt gaan gebruiken om de objecten van je schets werkelijk te maken

2.Waarop moet je letten als je een breakdown schets maakt Een duidelijke onderscheidt maken tussen de elementen en properties van de verschillende talen. dit kan met verschillende kleuren.

Daily checkout

3 dingen die ik geleerd heb:

- Belangen breakdownschets
- css-animaties
- css-color-gradient

Ik vind breakdown-schetsen, CSS-animaties en CSS color gradients echt leuk, en ik wil ze graag gebruiken voor mijn visitekaartje-project. Eerst wil ik een breakdown-schets maken om de indeling van mijn visitekaartje goed te plannen. Ik ben vooral enthousiast om een animatie toe te voegen, zoals een subtiele beweging of een effect wanneer je de pagina opent. Ook wil ik met CSS color gradients mooie kleurovergangen maken om het visitekaartje er speciaal uit te laten zien. Deze technieken maken mijn visitekaartje niet alleen nuttig, maar ook aantrekkelijk en levendig.

2 dingen die ik interesant vind:

- css-animaties
- css-flexbox

Ik vind CSS-animaties en CSS-flexbox echt leuk. Met CSS-animaties kan ik elementen op een pagina laten bewegen en veranderen, wat mijn ontwerpen levendiger maakt. Het is spannend om te experimenteren met verschillende bewegingen en effecten. CSS-flexbox vind ik ook interessant, maar ik heb gemerkt dat het best wel lastig kan zijn om mee te werken. Het helpt me wel om elementen netjes en flexibel te plaatsen, maar het kan soms ingewikkeld zijn om alles goed uit te lijnen. Ik blijf oefenen om het beter onder de knie te krijgen.
